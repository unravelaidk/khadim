## Design Context

### Users
Khadim serves three tiers: **simple automators** (office workers describing tasks in English), **builders** (IT/automation engineers configuring multi-step agents), and **deployers** (enterprise platform leads managing fleets). The Work UI primarily serves builders and deployers — people who need to see what agents are doing, intervene when needed, and trust the system to run reliably.

### Brand Personality
**Easy · Modern · Unhurried**

Khadim should feel like a place you want to spend time. Calm confidence — the user trusts what they see, never feels rushed or overwhelmed. The interface recedes so the work comes forward.

### Aesthetic Direction
**References**: Linear (density with clarity, crafted feel, minimal chrome), Vercel (quietness, restraint, typographic confidence). Both share: strong type hierarchy doing the heavy lifting, monochrome palettes with surgical color, generous whitespace that signals control rather than emptiness.

**Anti-references**: Cluttered admin dashboards (UiPath, Retool). Anything that looks like a generic SaaS control panel — tables everywhere, metric cards in a grid, identical repeated components stamped from a template.

**Theme**: Light (default). Near-white neutral surfaces, near-black text, minimal chrome. Dark mode remains available via the footer toggle, but the product's primary aesthetic is a calm, airy light surface that sits well next to productivity tools and documents. Inspired by conversational home surfaces (ChatGPT-style): soft-grey interactive states, rounded rails, typography-led hierarchy.

### Design Principles
1. **Each view earns its shape.** No two list views should be structural clones. The agent list, session list, and credential list serve different mental models — they should look different.
2. **Typography is the design.** Type scale, weight, and spacing create hierarchy — not borders, cards, or colored containers. Use fewer sizes with more contrast between them.
3. **Color is signal, not decoration.** Status colors (running, error, success) are the only non-neutral colors in the Work UI. They mean something. Everything else is neutral.
4. **Calm density.** Show enough information to build confidence, but not so much that the user scans instead of reads. Generous vertical rhythm between sections.
5. **Progressive disclosure.** Start quiet. Reveal detail on hover, click, or expand. The resting state should feel clean.
