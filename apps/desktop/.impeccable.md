## Design Context

### Users
Khadim serves three tiers: **simple automators** (office workers describing tasks in English), **builders** (IT/automation engineers configuring multi-step agents), and **deployers** (enterprise platform leads managing fleets). The Work UI primarily serves builders and deployers — people who need to see what agents are doing, intervene when needed, and trust the system to run reliably.

### Brand Personality
**Easy · Modern · Unhurried**

Khadim should feel like a place you want to spend time. Calm confidence — the user trusts what they see, never feels rushed or overwhelmed. The interface recedes so the work comes forward.

### Aesthetic Direction
**References**: Linear (density with clarity, crafted feel, minimal chrome), Vercel (quietness, restraint, typographic confidence). Fitness/social card UIs (shadow-driven depth, pillowy surfaces, tinted fills per category, personality through varied card shapes). Travel itinerary UIs (strong typographic hierarchy, timeline structure, physical paper-on-surface feel).

Core qualities drawn from these references:
- **Shadow-driven depth** — surfaces feel physical. Cards float above the background through soft multi-layered shadows, not borders. Borders are used sparingly or not at all.
- **Tinted personality** — each category/type gets its own subtle color wash (rose, amber, lime, sky, violet, teal). Color is applied through filled surfaces, not outlines.
- **Pillowy generosity** — large border radius (20px+), generous padding, round icon containers. Nothing feels cramped.
- **Bold confident buttons** — solid dark fills for primary actions (btn-ink), not ghost/glass. Secondary actions as outline pills or text.
- **Each component earns its own shape** — connected cards ≠ catalog cards ≠ search well ≠ connect sheet.

**Anti-references**: Cluttered admin dashboards (UiPath, Retool). Anything that looks like a generic SaaS control panel — tables everywhere, metric cards in a grid, identical repeated components stamped from a template. Flat glass-on-glass with visible borders on everything.

**Theme**: Light (default). Near-white neutral surfaces, near-black text, minimal chrome. Dark mode remains available via the footer toggle, but the product's primary aesthetic is a calm, airy light surface that sits well next to productivity tools and documents.

**Depth system** (CSS tokens):
- `depth-card` / `depth-card-sm` — raised surfaces with soft drop shadows, no borders
- `depth-card-interactive` — raised cards that lift on hover with shadow growth
- `depth-inset` — recessed surfaces (inputs, embedded content)
- `depth-well` — deeply recessed containers (search, empty states)
- `--tint-*` variables — per-category color washes for personality

### Design Principles
1. **Each view earns its shape.** No two list views should be structural clones. The agent list, session list, and credential list serve different mental models — they should look different.
2. **Shadow is the new border.** Depth through layered shadows, not outlines. Cards should feel like physical objects on a surface. Use `depth-card`, `depth-inset`, `depth-well` instead of `glass-card` + borders where appropriate.
3. **Color is personality, not just signal.** Status colors (running, error, success) signal state. But tinted fills (`--tint-*`) give categories, types, and sections their own warm identity. The monochrome base earns its calm from contrast with these intentional color moments.
4. **Typography is the design.** Type scale, weight, and spacing create hierarchy — not containers. Use fewer sizes with more contrast between them.
5. **Calm density.** Show enough information to build confidence, but not so much that the user scans instead of reads. Generous vertical rhythm between sections.
6. **Progressive disclosure.** Start quiet. Reveal detail on hover, click, or expand. The resting state should feel clean.
